## 2025-01-20 - Fix Command Option Injection Vulnerability
**Vulnerability:** Shell scripts or command lines using external commands (like `compare` in ImageMagick) are vulnerable to option injection when given unvalidated filenames that start with a hyphen (`-`). For instance, a file named `-help` could trigger an unintended argument, resulting in arbitrary options passed to the external program rather than being interpreted as a filename.
**Learning:** Even when inputs are properly quoted or escaped (e.g., using `escapeShellArg`), option injection can still occur because the target executable interprets the first character as a command-line flag rather than a positional file argument.
**Prevention:** Always sanitize or prepend user-controlled or external file paths with a safe path prefix, such as `./`, to explicitly denote that the argument is a file path and not a command-line flag. A simple `safeFilePath` helper function that checks if a path starts with `-` and prepends `./` is sufficient to mitigate this vulnerability.
