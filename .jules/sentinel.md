## 2024-05-18 - Option Injection via popen()
**Vulnerability:** Command option injection vulnerability in `CompareWithReference` and `CompareLocalImages` via unsanitized input concatenated into shell commands executed using `popen()`. Although `escapeShellArg` properly escapes quotes, if a filename starts with `-`, ImageMagick `compare` interprets it as a command flag, leading to potential execution of unintended logic.
**Learning:** Even internal test tools can be vulnerable if they execute shell commands using unvalidated strings (like file paths or image names). Proper shell escaping (`'...'`) does not prevent option injection. Prepending `./` to relative filenames fixes this for many CLI tools.
**Prevention:** Avoid `popen` and `system`. If shell execution is strictly necessary, safely format file paths by prepending `./` to prevent them from being parsed as flags, and use APIs like `execve` which don't invoke a shell and pass arguments in an array instead of concatenating them.
