## 2026-03-26 - [CRITICAL] Fix Option Injection in ImageMagick `compare` Command
**Vulnerability:** Option Injection in `popen` command execution. When creating shell commands using user-controlled input (like file paths for `actual`, `referenceName`), an attacker could craft a filename starting with `-` (e.g., `-someoption`). Even with `escapeShellArg` escaping spaces and quotes, tools like ImageMagick will treat files starting with `-` as command-line options, potentially leading to arbitrary file read/write or command execution via tool-specific features.
**Learning:** Shell quoting (`escapeShellArg`) is not sufficient when arguments are passed to tools that interpret `-` prefixed strings as options. This is a common pattern in CI test tools that spawn shell commands.
**Prevention:** Always ensure that dynamically generated file paths passed to shell commands start with an explicit directory path (like `./` or `/`) to force the underlying tool to interpret the string as a path, not an option. A `safeFilePath` helper prepending `./` to relative paths provides this guarantee.
